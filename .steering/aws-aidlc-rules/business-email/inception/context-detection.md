# コンテキスト検出ルール

## 目的

ユーザーがすでに提供している情報量を評価し、続けて実行すべきインセプションステージを決定する。

---

## 検出チェックリスト

ユーザーの入力を以下の5つの次元で評価する：

| 次元 | 確認内容 | 状態 |
|------|----------|------|
| **目的** | メールの目標が明確か（依頼 / 報告・通知 / 謝罪 / 交渉 / フォローアップ / 自己紹介） | [ ] 判明 / [ ] 不明 |
| **受信者** | 受信者の立場と送信者との関係が判明しているか | [ ] 判明 / [ ] 不明 |
| **要点** | 伝えるべき主要ポイントが特定されているか | [ ] 判明 / [ ] 不明 |
| **トーン** | 適切なトーンが明確か（フォーマル / 標準ビジネス / カジュアル） | [ ] 判明 / [ ] 不明 |
| **制約** | 制約事項があるか（言語・文字数・期限・添付など） | [ ] 判明 / [ ] 不明 |

---

## 判断ロジック

```
5/5 次元が判明している場合：
    → フェーズ2（コンストラクション）へ進む

3〜4/5 次元が判明している場合：
    → 不明な次元のみを対象にステージ1.2（要件分析）を実行

0〜2/5 次元が判明している場合：
    → インセプション全体を実行（ステージ1.2 → 1.3 → 1.4 を適宜実施）
```

---

## 出力

検出結果をセッション状態に記録する：

```markdown
## コンテキスト検出結果
- 目的：[判明 / 不明]
- 受信者：[判明 / 不明]
- 要点：[判明 / 不明]
- トーン：[判明 / 不明]
- 制約：[判明 / 不明]
- 判断：[コンストラクションへ直行 / 部分的な要件収集 / インセプション全体実行]
```
