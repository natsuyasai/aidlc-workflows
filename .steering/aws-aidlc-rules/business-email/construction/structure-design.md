# 構造設計ルール

## 目的

本文を書く前にメールの骨格を定義し、論理的な流れと適切な長さを確保する。

---

## 日本語ビジネスメールの標準構成

```
【件名】
  → 明確・具体的、アクションが必要な場合は行動を促す表現

【宛名】
  → 株式会社〇〇 〇〇部 〇〇様
  → 〇〇さん（社内セミフォーマル）

【書き出し】
  → 関係性を示す定型句
  → 自己紹介（初対面のみ）
  → なぜ連絡しているかを1文で

【本文】
  → 主要メッセージ・ブロック1
  → 主要メッセージ・ブロック2（必要な場合）
  → 補足情報・詳細

【依頼・アクション】
  → 求めることを明示
  → 期限がある場合は明記

【結び】
  → 丁寧な結びの表現
  → 問い合わせを歓迎する一言

【署名】
  → 氏名 / 所属 / 連絡先
```

---

## 英語ビジネスメールの標準構成

```
【件名】
  → 明確・簡潔・行動を促す表現

【書き出しの挨拶】
  → Dear [Name], / Hi [Name], / Hello [Name],

【冒頭文】
  → 最初の1〜2文でメールの目的を明示

【本文】
  → 重要な情報を論理的な順序で
  → 複数の項目は箇条書き

【アクション要求】
  → 明確な次のステップと期限

【結び】
  → Thank you / Best regards / Kind regards

【署名】
  → 氏名 / 役職 / 会社名 / 連絡先
```

---

## 長さの目安

| メールの種類 | 目標文字数（日本語）|
|------------|------------------|
| 簡単な返信・確認 | 100字以内 |
| 標準的な依頼・報告 | 100〜250字 |
| 複数要点を含む複雑なメール | 250〜450字 |
| 正式な提案・詳細報告 | 450字以上（できるだけ添付ファイルを活用） |

**原則**：必要なことは漏れなく、不要なことは一切書かない。短い方が良い。

---

## 出力

完成した文章ではなく、骨格アウトラインを作成してドラフト生成ステージへ引き渡す。
