# 記事構成計画ルール

## 目的

執筆開始前に記事全体の骨格（タイトル・章立て・コード例の配置・まとめ）を設計し、執筆をスムーズに進めるための青写真を作る。

---

## 計画テンプレート

```markdown
## 記事構成計画

### タイトル候補
1. [候補1 — 技術名・解決する課題を含むもの]
2. [候補2 — 読者の「知りたいこと」に直接答えるもの]
3. [候補3 — 体験・発見を前面に出したもの（体験談記事向け）]

### リード文（冒頭フック）
- アプローチ：[課題提示 / 驚き・発見 / 読者への問いかけ / 結論先出し]
- 伝える内容：[この記事で何が得られるかを1〜2文で]

### 目次（章立て）
1. [セクション1：背景・課題の説明]
2. [セクション2：概念・仕組みの説明]
3. [セクション3：実装・手順（コアコンテンツ）]
4. [セクション4：応用・発展・注意点]
5. [セクション5：まとめ・次のステップ]

### コード例の配置
- [セクション3：動作確認用の最小コード例]
- [セクション4：実用的な拡張コード例]

### まとめセクションの構成
- 記事の要点（3点以内）
- 読者への次のステップ（CTA）
- 参考リソース・関連記事リンク

### 想定ボリューム
- [短め（2,000字以内）/ 標準（2,000〜5,000字）/ 長め（5,000字以上）]
- 見出し数：[〇個]、コードブロック数：[〇個]
```

---

## 記事種別ごとの推奨構成

### チュートリアル
```
1. はじめに（何を作るか・完成イメージ）
2. 前提条件・環境構築
3. ステップ1：〇〇する
4. ステップ2：〇〇する
5. ステップ3：〇〇する
6. 動作確認
7. まとめ・発展的な内容
```

### 技術解説（Deep Dive）
```
1. はじめに（なぜこの技術を解説するか）
2. 概要・全体像
3. 内部構造・仕組みの解説
4. 実装例・コード解説
5. パフォーマンス特性・注意点
6. まとめ・参考資料
```

### 比較・選定記事
```
1. はじめに（比較の背景・選定の必要性）
2. 比較軸の定義
3. 選択肢Aの評価
4. 選択肢Bの評価
5. （選択肢Cがある場合）
6. 総合比較表
7. どのケースで何を選ぶべきか
8. まとめ
```

### 体験談・振り返り
```
1. はじめに（状況・課題の背景）
2. 試みたこと・失敗した経緯
3. 転換点・気づき
4. 解決策・採用したアプローチ
5. 結果・学んだこと
6. まとめ・読者へのメッセージ
```

---

## タイトルの書き方ガイド

| 記事種別 | パターン | 例 |
|---------|----------|-----|
| チュートリアル | 〜を〇分でやってみた / 〜入門 | RustでWebサーバーを30分で作る |
| 技術解説 | 〜の仕組みを徹底解説 / 〜の内部実装を読んだ | GoのGCアルゴリズムを徹底解説 |
| 比較 | 〜と〜を比較してみた / 〜を選ぶ際の判断基準 | PostgreSQLとMySQLを本番導入前に比較した |
| 体験談 | 〜してみてわかったこと / 〜で詰まった話 | 大規模リファクタリングを完走してわかったこと |
| 意見 | 〜について考えてみた / 〜に思うこと | AIコード生成ツールと1年付き合って思うこと |

---

## 出力

記事構成計画はステージ2.1（構造設計）に直接引き渡し、ドラフト作成の青写真として使用する。
