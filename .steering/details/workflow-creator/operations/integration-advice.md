# 統合アドバイスルール

## 目的

生成したワークフロー定義ファイルをリポジトリに組み込み、
実際に Claude Code で使える状態にするための手順と推奨事項を提供する。

---

## 統合手順

### ステップ1：ファイルの配置確認

生成されたファイルが以下の場所に正しく配置されていることを確認する：

```
aidlc-rules/[作業名]/          ← コアワークフロー（core-workflow.md のみ）
details/[作業名]/              ← 詳細ルールファイル群
aidlc-docs/[作業名]/           ← ドキュメント・状態管理
```

### ステップ2：CLAUDE.md への追記

`CLAUDE.md` の「有効なワークフロー」セクションに以下を追記する：

```markdown
- **[作業の表示名]** — `aidlc-rules/[作業名]/core-workflow.md`
```

あわせて「ドキュメント構成」セクションに以下を追記する：

```markdown
├── [作業名]/
│   ├── aidlc-state.md       # 現在のセッション状態
│   ├── audit.md             # セッション監査ログ
│   └── [成果物フォルダ名]/  # 生成した成果物
```

### ステップ3：動作確認

以下の手順でワークフローが正しく機能するか確認する：

1. Claude Code セッションを新たに開始する
2. 対象作業の依頼を行う（例：「[作業名]をしてほしい」）
3. インセプションフェーズが正しく始まり、必要な質問が行われることを確認する
4. 情報を提供し、コンストラクションフェーズへ移行することを確認する
5. 成果物が要求した形式で出力されることを確認する

---

## ワークフロー品質の継続的改善

### 改善が必要なサインとその対処

| 症状 | 考えられる原因 | 対処方法 |
|------|-------------|---------|
| 毎回同じ質問を手動で追加している | requirements-analysis.md の質問バンクが不足 | 質問バンクに追加する |
| スキップされるべきステージが実行される | context-detection.md の判断ロジックが不明確 | 閾値・条件を見直す |
| 成果物の品質にばらつきがある | 生成ルールの指針が曖昧 | 具体例・フレーズ例を追加する |
| レビューチェックで見逃しが多い | review.md のチェックリストが不足 | 実際に見逃したケースを追加する |
| 作業の途中で方針変更が多い | インセプションで引き出す情報が不足 | requirements-analysis.md の質問を見直す |

### バージョン管理の推奨

ワークフロー定義ファイルを改善する際：
- 変更前にファイルの内容を `aidlc-docs/[作業名]/` にバックアップする
- 変更内容と理由を `aidlc-docs/[作業名]/audit.md` に記録する

---

## 新しいワークフローを追加する際の注意点

| 確認事項 | 理由 |
|---------|------|
| `CLAUDE.md` の行数が増えすぎていないか（200行以内を目安） | CLAUDE.md は常にコンテキストに読み込まれるため、長すぎると非効率 |
| 既存ワークフローと名前が衝突していないか | ディレクトリ名・表示名の重複を避ける |
| 類似する既存ワークフローの参照元を明示しているか | 設計の一貫性を保ちやすくなる |

---

## 出力

ユーザーが求めた場合のみ、簡潔な箇条書きで統合アドバイスを提供する：

```
【統合アドバイス】
- CLAUDE.md への追記：[追記すべき行]
- 動作確認手順：[確認ステップの要約]
- 推奨改善サイクル：[次回の見直しタイミング]
```
