# コンテキスト検出ルール

## 目的

ユーザーがすでに提供している情報量を評価し、続けて実行すべきインセプションステージを決定する。

---

## 検出チェックリスト

ユーザーの入力を以下の6つの次元で評価する：

| 次元 | 確認内容 | 状態 |
|------|----------|------|
| **テーマ** | 記事で扱う技術・トピックが明確か | [ ] 判明 / [ ] 不明 |
| **記事種別** | どのタイプの記事かが判明しているか（チュートリアル / 解説 / 体験談 / 比較 / 意見 / ニュース解説） | [ ] 判明 / [ ] 不明 |
| **読者層** | 想定読者のスキルレベルが判明しているか（入門 / 中級 / 上級） | [ ] 判明 / [ ] 不明 |
| **要点** | 読者に伝えたい主要なポイントが特定されているか | [ ] 判明 / [ ] 不明 |
| **コード例** | コードサンプルの要否・使用言語が判明しているか | [ ] 判明 / [ ] 不明 |
| **制約** | 制約事項があるか（文字数・公開先・締め切り・シリーズ記事かどうかなど） | [ ] 判明 / [ ] 不明 |

---

## 判断ロジック

```
5〜6/6 次元が判明している場合：
    → ステージ1.3（読者・目的分析）から開始

3〜4/6 次元が判明している場合：
    → 不明な次元のみを対象にステージ1.2（要件分析）を実行

0〜2/6 次元が判明している場合：
    → インセプション全体を実行（ステージ1.2 → 1.3 → 1.4）
```

---

## 出力

検出結果をセッション状態に記録する：

```markdown
## コンテキスト検出結果
- テーマ：[判明 / 不明]
- 記事種別：[判明 / 不明]
- 読者層：[判明 / 不明]
- 要点：[判明 / 不明]
- コード例：[判明 / 不明]
- 制約：[判明 / 不明]
- 判断：[1.3から開始 / 部分的な要件収集 / インセプション全体実行]
```
