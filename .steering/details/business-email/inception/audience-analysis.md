# 受信者・トーン分析ルール

## 目的

受信者のコンテキストを分析し、社外メールや配慮が必要なメールでトーンのミスマッチを防ぐために適切なコミュニケーションスタイルを決定する。

---

## 受信者プロファイル

受信者の状況を以下の観点で整理する：

| 要素 | 確認内容 |
|------|----------|
| **立場・役職** | 送信者より上位・同等・下位のいずれか |
| **関係性** | 初対面か、継続的な取引関係か |
| **文化的背景** | 日本のビジネス慣行 / 国際的 / 混合 |
| **感情的状況** | 配慮が必要な場面か（苦情・遅延・悪い知らせなど） |
| **専門知識** | 技術的・専門的な用語の理解度はどの程度か |

---

## トーンレベル

適切なトーンレベルを選択する：

```
レベル1 — 非常にフォーマル
  適用場面：役員・経営層への初連絡、公式な苦情、法務・コンプライアンス関連
  特徴：謹啓/拝啓・敬具、尊敬語・謙譲語を徹底使用、略語なし

レベル2 — 標準ビジネス
  適用場面：ほとんどの社外ビジネス文書、クライアントへの連絡、取引先とのやりとり
  特徴：「お世話になっております」、「ご確認のほどよろしくお願いいたします」

レベル3 — セミフォーマル
  適用場面：なじみのあるクライアント、頻繁にやりとりする協力会社、他部門との社内連絡
  特徴：丁寧だが敬語は軽め、適度な文章量

レベル4 — カジュアル社内
  適用場面：日常的なチーム連絡、簡単な進捗報告、気心の知れた同僚
  特徴：簡潔な文章、冒頭の形式的な挨拶は不要
```

---

## センシティビティフラグ

以下のいずれかに該当する場合は特別な配慮が必要：

- [ ] 悪い知らせを伝える（遅延・断り・ミスの報告）
- [ ] 異例または重要度の高い依頼をする
- [ ] 返信がないメールへのフォローアップ（相手が気まずい可能性）
- [ ] 謝罪またはミスの承認
- [ ] 重要なステークホルダーへの第一印象となるメール

**フラグが立った場合**：要件ブリーフにセンシティビティを記載し、ドラフトに緩和表現を適用する（共感フレーズ・問題の明確な引き受け・解決策重視の表現）。

---

## 出力

要件ブリーフに以下を追記する：

```markdown
- **受信者レベル**：[レベル1〜4・選定理由]
- **センシティビティフラグ**：[一覧 または「なし」]
- **トーン補足**：[ドラフト作成に向けた具体的な指針]
```
