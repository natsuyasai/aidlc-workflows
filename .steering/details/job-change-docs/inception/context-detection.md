# コンテキスト検出ルール

## 目的

ユーザーがすでに提供しているキャリア情報量を評価し、
続けて実行すべきインセプションステージを決定する。

---

## 検出チェックリスト

ユーザーの入力を以下の6つの次元で評価する：

| 次元 | 確認内容 | 状態 |
|------|----------|------|
| **現職の役割** | 職種・ポジション・担当領域が判明しているか | [ ] 判明 / [ ] 不明 |
| **主要な実績** | 具体的な成果・担当プロジェクト・貢献が判明しているか | [ ] 判明 / [ ] 不明 |
| **スキル・技術スタック** | 保有スキル・使用技術・専門領域が判明しているか | [ ] 判明 / [ ] 不明 |
| **転職目的** | なぜ転職するのか・何を変えたいのかが判明しているか | [ ] 判明 / [ ] 不明 |
| **ターゲット職種** | 目指す役割・職種・ポジションレベルが判明しているか | [ ] 判明 / [ ] 不明 |
| **作成する書類種別** | 職務経歴書・自己PR・いずれも必要かが判明しているか | [ ] 判明 / [ ] 不明 |

---

## 判断ロジック

5〜6/6 次元が判明している場合：
    → ステージ1.3（キャリア実績分析）から開始

3〜4/6 次元が判明している場合：
    → 不明な次元のみを対象にステージ1.2（要件分析）を実行

0〜2/6 次元が判明している場合：
    → インセプション全体（ステージ1.2〜1.4）を実行

---

## 出力

検出結果をセッション状態に記録する：

```markdown
## コンテキスト検出結果（ステージ1.1）

- 現職の役割：[判明 / 不明]
- 主要な実績：[判明 / 不明]
- スキル・技術スタック：[判明 / 不明]
- 転職目的：[判明 / 不明]
- ターゲット職種：[判明 / 不明]
- 作成する書類種別：[判明 / 不明]

→ 判断：[次のアクション]
```
