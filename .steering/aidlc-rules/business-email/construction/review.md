# レビュー・洗練ルール

## 目的

ユーザーに提示する前に生成したドラフトを要件に照らして自己レビューし、フィードバックに基づく反復的な改善をガイドする。

---

## 提示前チェックリスト

すべてのドラフトを提示する前にこのチェックリストを実行する：

### 内容
- [ ] メールの目的が最初の2文以内で明確になっている
- [ ] 要件ブリーフのすべての要点が含まれている
- [ ] アクション要求（ある場合）が明確で曖昧さがない
- [ ] 重要な情報の抜けがない

### トーン・言語
- [ ] トーンがレベル1〜4の目標水準で全体を通じて一貫している
- [ ] 敬語レベルが適切かつ一貫している（日本語の場合）
- [ ] セクション間でトーンの急変がない

### 明確さ
- [ ] 曖昧な時期表現がない（具体的な日付に置き換えていること）
- [ ] 専門用語が受信者の知識レベルに適している
- [ ] 1文が過度に長くない（日本語：1文60字以内を目安、英語：1文25語以内）

### 長さ
- [ ] メールが目標の長さの範囲内に収まっている
- [ ] 不要な空白表現や重複した文がない

### 構成
- [ ] 件名が内容を正確に表している
- [ ] 書き出しが適切に文脈を設定している
- [ ] 結びに明確な次のステップが示されている

---

## 洗練のガイダンス

ユーザーから修正依頼を受けた場合、フィードバックを分類して対応する：

| フィードバックの種類 | 対応方法 |
|---------------------|----------|
| 「もっとフォーマルに」 | トーンレベルを1段階上げ、くだけた表現を敬語に置き換える |
| 「もっと短く」 | 冒頭の形式的な挨拶を削り、本文を凝縮し、結びを簡潔にする |
| 「もっと強く依頼したい」 | 依頼を明確にし、具体的な期限を追加し、より強い動詞を使う |
| 「ポイントを追加したい」 | 既存の構成に自然に組み込み、長さを再確認する |
| 「全体的に書き直したい」 | 更新された要件でドラフト生成に戻る |

---

## ユーザーへの提示方法

チェックリストを通過したら、以下の形式でドラフトを提示する：

1. **ドラフト本体** — クリーンなコードブロック内に記載
2. **作成メモ** — ドラフト作成中に設けた前提（最大3点）
3. **修正の案内** — フィードバックを促す一言

例：

```
以下のドラフトを作成しました。

---
[ドラフト]
---

【作成メモ】
- 締め切りが未指定のため「今週中」としました
- トーン：レベル2（標準ビジネス）を適用
- 添付ファイルへの言及は省略しました（必要であればお知らせください）

修正希望があればお知らせください。
```
