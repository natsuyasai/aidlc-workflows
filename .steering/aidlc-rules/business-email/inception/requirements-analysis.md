# 要件分析ルール

## 目的

コンテキスト検出で「不明」と判定された次元について、構造化された質問で必要な情報を収集する。

---

## 質問バンク

コンテキスト検出で「不明」とされた次元についてのみ質問する。一度に質問するのは最大5問まで。

### 目的に関する質問
- このメールの目的は何ですか？
  - 依頼 / 報告・通知 / 謝罪 / 交渉・提案 / フォローアップ / 自己紹介・挨拶
- 相手にどのようなアクションをとってほしいですか？（返信 / 承認 / 情報提供 / 特になし）

### 受信者に関する質問
- 受信者はどのような立場の方ですか？
  - 社内の上司 / 同僚 / 部下 / 社外のクライアント / 初対面の取引先 / その他
- 相手との関係性・これまでのやりとりの経緯はありますか？

### 要点に関する質問
- メールで伝えたい主要なポイントを箇条書きで教えてください
- 特に強調したいこと・必ず含めたいことはありますか？

### トーンに関する質問
- どのくらいのフォーマルさが適切ですか？
  - 非常にフォーマル（初対面・重要案件）/ 標準的なビジネス敬語 / カジュアルな社内連絡

### 制約に関する質問
- メールの言語は何語ですか？（日本語 / 英語 / その他）
- 文字数や長さの目安はありますか？
- 添付ファイルへの言及は必要ですか？
- 返信期限や送付希望日時はありますか？

---

## 出力形式

回答を収集したら、**要件ブリーフ**としてまとめる：

```markdown
## メール要件ブリーフ

- **目的**：[目的]
- **受信者**：[立場 / 関係性]
- **要点**：
  1. [ポイント1]
  2. [ポイント2]
- **トーン**：[フォーマルレベル]
- **言語**：[言語]
- **制約**：[文字数 / 期限 / 添付など]
```

この要件ブリーフをコンストラクションフェーズへの正式なインプットとして引き渡す。

---

## 適応的な深度

| 状況 | 対応 |
|------|------|
| 短い社内メール | 2〜3問に絞って質問 |
| 社外クライアントへの初回コンタクト | 関連するすべての質問を実施 |
| 「とりあえずドラフトして」と言われた場合 | 合理的なデフォルト値を適用し、ブリーフに前提として記載 |
