# ターゲット分析ルール

## 目的

ターゲット企業・職種・求人票の要件を分析し、
ユーザーのキャリア実績との適合ポイントと差分を特定する。

---

## 分析の観点

### 1. 求人要件の読み解き

求人票・JD（Job Description）が提供された場合、以下を抽出する：

| 要素 | 確認内容 |
|------|---------|
| **必須スキル** | MUST条件として明記されているスキル・経験 |
| **歓迎スキル** | WANT条件・プラス評価される経験 |
| **役割・責任** | ポジションが担う役割と期待されるアウトプット |
| **チーム・組織文化** | チーム構成・開発スタイル・価値観 |
| **ポジションレベル** | シニア・テックリード・アーキテクト等の期待値 |

求人票がない場合は、ターゲット職種の一般的な要件をもとに分析する。

### 2. 適合ポイントの特定

キャリア実績分析の結果と照合し、アピールすべき強みを特定する：

```
[強み] → [求人要件への対応]
例：
- 複数チームへの技術フォロー経験 → 「横断的技術支援が求められる役割」への適合
- 〇〇技術でのリード経験 → 必須スキルへの完全適合
- 大規模サービスでのパフォーマンス改善 → 歓迎スキルへの対応
```

### 3. 差分の特定と対処方針

求人要件とのギャップがある場合：

| ギャップの種類 | 対処方針 |
|-------------|---------|
| **完全に未経験の必須スキル** | 正直に認めつつ習得意欲・類似経験で補う |
| **経験が浅い歓迎スキル** | 関連する実績を前景化し、学習中であれば明示 |
| **ポジションレベルのギャップ** | 実績の規模感・リード経験で補強する |

### 4. 訴求メッセージの設計

ターゲット企業・職種に対して「なぜ自分が適任か」を1〜2文で設計する：

```
[ポジション名]として、[強み1]と[強み2]を活かし、[企業・チームへの貢献]を実現します。
```

---

## 出力

分析結果をセッション状態に記録する：

```markdown
## ターゲット分析結果（ステージ1.4）

### ターゲット情報
- 企業・職種：[名称または「未定」]
- ポジションレベル：[シニア / テックリード / アーキテクト / EM など]

### 適合ポイント
- [強み] → [対応する要件]
- [強み] → [対応する要件]

### 差分・補完が必要な点
- [ギャップ]：[対処方針]

### 訴求メッセージ（ドラフト）
[1〜2文の訴求メッセージ]
```
