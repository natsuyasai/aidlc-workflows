# レビュー・洗練ルール

## 目的

生成したワークフロー定義ファイル群を横断的にレビューし、
一貫性・網羅性・実用性の観点から品質を確認する。

---

## 提示前チェックリスト

### 網羅性（必要なファイルがすべて揃っているか）
- [ ] `core-workflow.md` が生成されている
- [ ] `inception/context-detection.md` が生成されている
- [ ] `inception/requirements-analysis.md` が生成されている
- [ ] ファイル構成計画で列挙した全ファイルが生成されている
- [ ] `construction/review.md` が生成されている
- [ ] `aidlc-docs/[作業名]/aidlc-state.md` が生成されている
- [ ] `CLAUDE.md` への追記内容が提示されている

### 一貫性（ファイル間で矛盾がないか）
- [ ] `core-workflow.md` のステージ一覧と実際に生成したファイルが対応している
- [ ] `core-workflow.md` のルールファイルパスがすべて正確（存在するファイルを指している）
- [ ] `context-detection.md` の次元と `requirements-analysis.md` の質問バンクが対応している
- [ ] `aidlc-state.md` の項目が `requirements-analysis.md` の要件ブリーフと対応している
- [ ] 適応的実行の条件が `core-workflow.md` と各ルールファイルで一致している

### 実用性（実際の作業で使えるか）
- [ ] コンテキスト検出の次元が多すぎない（3〜7個が適切）
- [ ] 質問バンクの質問が具体的で回答しやすい
- [ ] 生成ルールに具体的な例・テンプレートが含まれている
- [ ] レビューチェックリストの項目が作業の品質基準と対応している
- [ ] 適応的実行の表に実際によくあるケースが含まれている

### 文体・構造の統一
- [ ] すべてのファイルが日本語で書かれている
- [ ] 見出し構造が `business-email` / `tech-blog` と同じパターンに従っている
- [ ] 「目的」セクションが各ファイルの冒頭にある
- [ ] 「出力」セクションが各ファイルの末尾にある

---

## 洗練のガイダンス

| フィードバックの種類 | 対応方法 |
|---------------------|----------|
| 「ステージが多すぎる」 | 関連するステージを統合し、対応するルールファイルも統合する |
| 「この次元は不要」 | context-detection.md と requirements-analysis.md から削除し、state.md も更新 |
| 「このファイルの内容が薄い」 | 該当ファイルの生成テンプレートに戻り、具体例・フレーズ例を追加する |
| 「別のステージを追加したい」 | core-workflow.md → 対応ルールファイル → state.md の順で追加 |
| 「全体的に作り直したい」 | ファイル構成計画（ステージ1.4）に戻り、設計から見直す |

---

## ユーザーへの提示方法

全ファイルの生成が完了したら、以下の形式で提示する：

```
ワークフロー定義ファイルの生成が完了しました。

## 生成したファイル一覧
[ファイルパスの一覧]

## CLAUDE.md への追記内容
[追記すべき行]

## 使い方
このワークフローを使うには、[作業名]の作業を依頼する際に
Claude Code が自動的に `core-workflow.md` を読み込みます。

【レビューメモ】
- [前提・設計上の判断（最大3点）]

修正希望があればお知らせください。
```
