# 要件分析ルール

## 目的

コンテキスト検出で「不明」と判定された次元について、構造化された質問で必要な情報を収集する。

---

## 質問バンク

コンテキスト検出で「不明」とされた次元についてのみ質問する。一度に質問するのは最大5問まで。

### テーマに関する質問
- 記事で扱いたい技術・トピックを教えてください
- このテーマを取り上げようと思ったきっかけや背景はありますか？
  （業務での実体験 / 新技術のキャッチアップ / 問題を解決した方法の共有 / 社内知識の外部発信など）

### 記事種別に関する質問
- どのような形式の記事にしたいですか？
  - **チュートリアル**：手順に沿って読者が手を動かせる
  - **技術解説**：概念・仕組みを深く説明する
  - **体験談・振り返り**：実際の経験・失敗・学びを共有する
  - **比較・選定記事**：複数の技術・ツールを評価する
  - **意見・考察**：技術トレンドや設計思想について論じる
  - **ニュース解説**：新技術・新リリースの要点を解説する

### 読者層に関する質問
- 主な想定読者はどのようなエンジニアですか？
  - 入門者（プログラミング経験1年未満 / その技術を初めて触る）
  - 中級者（基本は理解しているが深掘りしたい）
  - 上級者（実務経験豊富、内部実装や設計判断に興味がある）
- 読者が記事を読む前に持っている前提知識はどの程度を想定していますか？

### 要点に関する質問
- この記事で読者に必ず伝えたいことを箇条書きで教えてください
- 記事を読み終えた読者に何をしてほしいですか？
  （手を動かしてほしい / 概念を理解してほしい / 自社ツールや手法を使ってみてほしい / 特になし）

### コード例に関する質問
- コードサンプルは必要ですか？必要な場合、使用する言語・フレームワークを教えてください
- コードの量はどの程度を想定していますか？
  （最小限のスニペット / セクションごとのサンプル / 動作する完全なサンプル）

### 制約に関する質問
- 記事の目標文字数や分量の目安はありますか？
- 公開先のプラットフォームは決まっていますか？（Zenn / Qiita / 社内ブログ / Medium / note / その他）
- 公開期限はありますか？
- シリーズ記事の一部ですか？前後の記事との関係はありますか？

---

## 出力形式

回答を収集したら、**要件ブリーフ**としてまとめる：

```markdown
## 技術ブログ 要件ブリーフ

- **テーマ**：[テーマ・技術名]
- **記事種別**：[種別]
- **読者層**：[スキルレベル・前提知識]
- **要点**：
  1. [ポイント1]
  2. [ポイント2]
- **読者への期待アクション**：[記事後に読者に何をしてほしいか]
- **コード例**：[要否・言語・量]
- **制約**：[文字数 / 公開先 / 締め切り / シリーズ情報]
```

この要件ブリーフをコンストラクションフェーズへの正式なインプットとして引き渡す。
